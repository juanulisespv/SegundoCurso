Mockups y Diseño Visual

Los mockups transforman wireframes en diseños visuales finales con colores, tipografía e identidad de marca.

Elementos del Diseño Visual

Color

Comunica emociones y jerarquía

  • Paleta primaria (2-3 colores)
  • Colores de acción (botones)
  • Estados (error, éxito, warning)
  • Accesibilidad (contraste WCAG)
Tipografía

Legibilidad y personalidad

  • Máximo 2-3 fuentes
  • Jerarquía (H1, H2, body)
  • Espaciado y line-height
  • Responsive (rem, em)
Espaciado

Organización y respiración visual

  • Sistema de 8pt grid
  • Márgenes consistentes
  • Whitespace intencional
  • Agrupación por proximidad
Iconografía

Comunicación visual rápida

  • Biblioteca consistente
  • Estilo uniforme (outline/fill)
  • Tamaños estandarizados
  • Significado universal
Ejemplo de evolución: Wireframe → Mockup

[Título]
[Descripción]
[Botón]

✨ MiApp

Gestiona tus tareas fácilmente

Sistemas de Diseño

Un sistema de diseño es una colección de componentes reutilizables y guías que aseguran consistencia en toda la aplicación.

¿Por qué Sistemas de Diseño?

  • Consistencia: Experiencia uniforme
  • Eficiencia: Reutilización de componentes
  • Escalabilidad: Crecer manteniendo coherencia
  • Colaboración: Lenguaje común diseño-desarrollo

Componentes de un Sistema de Diseño

Design Tokens

Valores fundamentales reutilizables

// Colores
primary: #007bff
secondary: #6c757d
success: #28a745

// Espaciado  
spacing-xs: 4px
spacing-sm: 8px
spacing-md: 16px
Componentes UI

Elementos reutilizables

  • Botones (primario, secundario)
  • Formularios (inputs, selects)
  • Navegación (navbar, breadcrumbs)
  • Feedback (alerts, tooltips)
Documentación

Guías de uso y patrones

  • Cuándo usar cada componente
  • Variaciones permitidas
  • Ejemplos de implementación
Implementación

Código reutilizable

  • CSS/SCSS variables
  • Componentes React/Vue
  • Storybook para documentar

Sistemas de Diseño Populares

Material Design (Google): Principios de diseño físico

Human Interface Guidelines (Apple): Estándares iOS/macOS

Fluent Design (Microsoft): Sistema Windows moderno

Prototipos Interactivos

Los prototipos interactivos permiten experimentar la interfaz antes de desarrollarla, validando flujos y comportamientos.

Tipos de Interactividad

Navegación Básica

Clics y transiciones

  • Links entre pantallas
  • Botones que cambian estado
  • Menús desplegables
Micro-interacciones

Feedback visual inmediato

  • Hover effects
  • Loading spinners
  • Animaciones de transición
Formularios Dinámicos

Validación en tiempo real

  • Validación mientras escribe
  • Campos condicionales
  • Autocompletado
Gestos Móviles

Interacciones táctiles

  • Swipe, pinch, scroll
  • Pull-to-refresh
  • Touch targets apropiados
Niveles de prototipado:
Clickable: Navegación básica entre pantallas
Functional: Formularios que validan, búsquedas que filtran  
Native: Prototipos que se sienten como app real
Coded: HTML/CSS funcional, casi implementación

Herramientas para Prototipos Interactivos

Herramientas de Diseño
  • Figma: Prototipos colaborativos
  • Adobe XD: Animaciones avanzadas
  • InVision: Especializado en prototipos
Código Interactivo
  • CodePen: Prototipos web rápidos
  • Framer: React components
  • ProtoPie: Prototipos de sensores

Fidelidad vs. Tiempo

Prototipos alta fidelidad toman mucho tiempo. Úsalos solo cuando necesites validar interacciones específicas o presentar a stakeholders importantes.

Testing de Usabilidad

El testing de usabilidad valida que el diseño funcione para usuarios reales en condiciones reales de uso.

Objetivos del Testing

  • Identificar problemas: Antes de desarrollar
  • Validar soluciones: ¿Resuelve el problema?
  • Optimizar flujos: Reducir fricción
  • Medir satisfacción: Experiencia del usuario

Métodos de Testing

Testing Moderado

Observación directa

  • 5-8 usuarios representativos
  • Tareas específicas
  • Think-aloud protocol
  • Observación de comportamiento
Testing No Moderado

Usuarios prueban independientemente

  • Herramientas online
  • Más participantes
  • Datos cuantitativos
  • Menos sesgos del moderador
A/B Testing

Comparar variaciones

  • Dos versiones simultáneas
  • Métricas de conversión
  • Decisiones basadas en datos
Testing Guerrilla

Testing rápido e informal

  • Cafeterías, espacios públicos
  • 15-20 minutos por sesión
  • Feedback rápido
Plan de testing típico:
Preparación: Definir tareas clave a evaluar
Reclutamiento: 5-8 usuarios del target
Sesiones: 45-60 min por usuario
Análisis: Identificar patrones de problemas
Reporte: Priorizar issues por severidad
Iteración: Rediseñar y volver a testear

Métricas de Usabilidad

Métricas de Eficiencia
  • Task completion rate: % tareas completadas
  • Time on task: Tiempo para completar
  • Clicks to complete: Eficiencia navegación
Métricas de Satisfacción
  • SUS Score: System Usability Scale
  • NPS: Net Promoter Score
  • CSAT: Customer Satisfaction

Handoff a Desarrollo

El handoff es la transferencia de diseños desde diseñadores a desarrolladores con toda la información necesaria para implementar.

Especificaciones de Diseño

Medidas y Espaciado
  • Dimensiones exactas (px, rem)
  • Márgenes y padding
  • Breakpoints responsive
  • Grillas y alineaciones
Especificaciones Visuales
  • Códigos de color (hex, RGB)
  • Fuentes y pesos
  • Sombras y efectos
  • Estados de interacción
Assets Exportados
  • Iconos (SVG, PNG)
  • Imágenes optimizadas
  • Diferentes resoluciones
  • Formato apropiado
Comportamientos
  • Animaciones y transiciones
  • Micro-interacciones
  • Estados de carga
  • Manejo de errores
Ejemplo de especificación CSS:
/* Botón Primario */
.btn-primary {
  background: #007bff;
  color: white;
  padding: 12px 24px;
  border-radius: 8px;
  font-weight: 600;
  transition: all 0.2s ease;
}

.btn-primary:hover {
  background: #0056b3;
  transform: translateY(-1px);
}

Comunicación continua es clave para implementar diseños fielmente:

  • Revisiones de implementación regulares
  • Sesiones de Q&A durante desarrollo
  • Ajustes colaborativos cuando necesario

Diseño de Flujos de Usuario

Los flujos de usuario mapean todos los pasos que un usuario debe seguir para completar una tarea en el sistema.

Componentes de un Flujo

  • Entry points: Cómo llega el usuario
  • Steps: Cada pantalla o acción
  • Decision points: Puntos de bifurcación
  • Exit points: Cómo termina el flujo
Ejemplo de flujo de compra:
Landing Page → Búsqueda → Producto → Carrito → Login/Register
     ↑              ↓          ↓         ↓           ↓
   Social       Categorías   Related   Checkout    Pago → Confirmación
   Media        Filtros      Items     Guest?      ↓         ↓
                                        ↓         Email    Tracking

Optimización de Flujos

Reducir Fricción
  • Minimizar pasos requeridos
  • Auto-completar información
  • Guest checkout options
  • Validación progresiva
Orientación Clara
  • Progress indicators
  • Breadcrumbs útiles
  • CTAs (Call-to-Action) obvios
  • Estados de proceso claros

Puntos de Abandono Comunes

  • Registro obligatorio: 70% abandona si requiere registro
  • Formularios largos: Más de 3 campos reduce conversión
  • Checkout complejo: 69% abandona carrito por complejidad

Resumen del Tema 5

Conceptos Clave Aprendidos

  • Diferencias entre UX y UI
  • Principios fundamentales de GUI
  • Wireframes y tipos de prototipos

El diseño GUI exitoso requiere proceso estructurado, validación constante con usuarios y colaboración estrecha entre diseño y desarrollo.

Puntos Clave para Recordar

  • Usuario primero: Diseñar para necesidades reales, no preferencias personales
  • Iterar temprano: Cambios baratos en wireframes, costosos en código
  • Validar constantemente: Testing frecuente evita sorpresas
  • Sistemas escalables: Componentes reutilizables facilitan crecimiento
