Modelado Arquitectónico de Sistemas

El modelado arquitectónico define la estructura general del sistema y cómo sus componentes interactúan para cumplir los requisitos.

¿Qué es la Arquitectura de Software?

Estructura organizacional del sistema que define componentes, relaciones y principios de diseño e implementación.

  • Componentes: Elementos funcionales del sistema
  • Conectores: Relaciones e interacciones entre componentes
  • Configuración: Topología de componentes y conectores

Importancia del Modelado Arquitectónico

Calidad del Software

Atributos de calidad definidos

  • Rendimiento y escalabilidad
  • Seguridad y disponibilidad
  • Mantenibilidad y modificabilidad
Comunicación Efectiva

Lenguaje común del equipo

  • Stakeholders entienden estructura
  • Decisiones arquitectónicas documentadas
  • Reducción de malentendidos

Aplicaciones Prácticas: Sistema Empresarial

Ejemplo completo aplicando todos los conceptos: herencia, polimorfismo, composición y arquitectura.

Sistema de gestión de empleados:
from abc import ABC, abstractmethod
from datetime import datetime

class Empleado(ABC):
    def __init__(self, nombre: str, email: str):
        self.nombre = nombre
        self.email = email
        self.fecha_ingreso = datetime.now()
    
    @abstractmethod
    def calcular_salario(self) -> float:
        pass

class EmpleadoTiempoCompleto(Empleado):
    def __init__(self, nombre: str, email: str, salario_base: float):
        super().__init__(nombre, email)
        self.salario_base = salario_base
        self.beneficios = BeneficiosCompletos()  # Composición
    
    def calcular_salario(self) -> float:
        return self.salario_base + self.beneficios.calcular()

class Gerente(EmpleadoTiempoCompleto):
    def __init__(self, nombre: str, email: str, salario_base: float, subordinados: int):
        super().__init__(nombre, email, salario_base)
        self.subordinados = subordinados
    
    def calcular_salario(self) -> float:
        base = super().calcular_salario()
        return base + (self.subordinados * 100)  # Bonus por subordinado

Conceptos Aplicados

  • Herencia: EmpleadoTiempoCompleto → Gerente
  • Polimorfismo: calcular_salario() específico por tipo
  • Composición: Empleados contienen BeneficiosCompletos
  • Abstracción: Clase base Empleado define interfaz común

Patrones de Diseño Aplicados

Factory Method para Empleados

Creación flexible:
class EmpleadoFactory:
    @staticmethod
    def crear_empleado(tipo: str, datos: dict) -> Empleado:
        if tipo == "tiempo_completo":
            return EmpleadoTiempoCompleto(
                datos["nombre"], datos["email"], datos["salario"]
            )
        elif tipo == "gerente":
            return Gerente(
                datos["nombre"], datos["email"], 
                datos["salario"], datos["subordinados"]
            )
        else:
            raise ValueError(f"Tipo no válido: {tipo}")

# Uso del Factory
empleado = EmpleadoFactory.crear_empleado("gerente", {
    "nombre": "Ana García", 
    "email": "ana@corp.com",
    "salario": 8000,
    "subordinados": 5
})

Singleton para Configuración

Configuración global:
class ConfiguracionEmpresa:
    _instance = None
    
    def __new__(cls):
        if cls._instance is None:
            cls._instance = super().__new__(cls)
            cls._instance.politicas = {
                "vacaciones_anuales": 21,
                "bonus_antiguedad": 0.05
            }
        return cls._instance
    
    def get_politica(self, nombre: str):
        return self.politicas.get(nombre)

# Uso global
config = ConfiguracionEmpresa()
vacaciones = config.get_politica("vacaciones_anuales")

Sistema Integrado

Demostración de todos los conceptos trabajando juntos en un sistema empresarial.

Sistema de RRHH completo:
class SistemaGestionRRHH:
    def __init__(self):
        self.empleado_factory = EmpleadoFactory()
        self.config = ConfiguracionEmpresa()  # Singleton
        self.departamentos = []
    
    def contratar_empleado(self, tipo: str, datos: dict) -> Empleado:
        # Factory crea empleado apropiado
        empleado = self.empleado_factory.crear_empleado(tipo, datos)
        return empleado
    
    def procesar_nomina(self, empleados: list) -> float:
        total = 0.0
        for empleado in empleados:
            # Polimorfismo: cada tipo calcula salario diferente
            salario = empleado.calcular_salario()
            total += salario
            print(f"{empleado.nombre}: ${salario:.2f}")
        return total

# Uso del sistema integrado
sistema = SistemaGestionRRHH()
empleados = [
    sistema.contratar_empleado("tiempo_completo", {
        "nombre": "Juan", "email": "juan@corp.com", "salario": 5000
    }),
    sistema.contratar_empleado("gerente", {
        "nombre": "María", "email": "maria@corp.com", 
        "salario": 8000, "subordinados": 3
    })
]

nomina_total = sistema.procesar_nomina(empleados)
print(f"Nómina total: ${nomina_total:.2f}")

Integración Completa

  • Herencia: Especialización progresiva de empleados
  • Polimorfismo: Procesamiento uniforme por tipo
  • Composición: Beneficios como componentes reutilizables
  • Patrones: Factory y Singleton trabajando en conjunto
