Introducción al Patrón Model-View-Controller (MVC)

El patrón MVC es una arquitectura de software que separa la lógica de aplicación en tres componentes principales para mejorar organización y mantenimiento.

¿Qué es el Patrón MVC?

Arquitectura que separa la aplicación en tres componentes principales:

  • Modelo (Model): Gestiona datos y lógica de negocio
  • Vista (View): Maneja presentación e interfaz de usuario
  • Controlador (Controller): Coordina Modelo y Vista

Beneficios de la Separación de Responsabilidades

Mantenimiento y Escalabilidad
  • Actualizar componentes sin afectar el resto
  • Agregar funcionalidades con menor impacto
  • Código más organizado y claro
Reutilización de Código
  • Vistas reutilizables con diferentes modelos
  • Controladores manejan múltiples vistas
  • Lógica de negocio independiente
Colaboración en Equipos
  • Backend trabaja en Modelo sin interferir Vista
  • Diseñadores modifican apariencia independientemente
  • Desarrollo paralelo de componentes
Mejor Organización
  • Separación reduce acoplamiento
  • Código más fácil de entender
  • Debugging y testing simplificados

Origen e Historia

El patrón MVC fue introducido en los años 1970 por Trygve Reenskaug en Xerox PARC, originalmente diseñado para mejorar la interacción usuario-sistema en entornos gráficos.

Componentes del MVC

El patrón MVC divide la estructura en tres componentes principales con responsabilidades específicas y bien definidas.

Usuario interactúa
         ↓
    [CONTROLADOR] ←→ [MODELO]
         ↓              ↓
       [VISTA] ←────────┘
         ↓
   Respuesta al Usuario

Modelo (Model) - Gestión de Datos y Lógica de Negocio

Responsabilidades del Modelo
  • Gestión de datos: Almacenar, recuperar y procesar información
  • Lógica de negocio: Implementar reglas y validaciones
  • Comunicación con BD: Manejo de persistencia
  • Notificación de cambios: Informar actualizaciones a vistas
Ejemplo en Python:
class Producto:
    def __init__(self, nombre, precio):
        self.nombre = nombre
        self.precio = precio
    
    def calcular_descuento(self, porcentaje):
        return self.precio * (1 - porcentaje / 100)
    
    def aplicar_iva(self, iva=21):
        return self.precio * (1 + iva / 100)

Vista (View) - Presentación de la Información

Responsabilidades de la Vista
  • Presentación de datos: Mostrar información de forma clara
  • Captura de interacciones: Formularios y eventos de usuario
  • Renderizado: HTML en web, GUI en aplicaciones escritorio
  • Interfaz de usuario: Experiencia visual y usabilidad
Template HTML (Flask):
<!-- productos.html -->
<div class="productos">
    {% for producto in productos %}
        <div class="producto-card">
            <h3>{{ producto.nombre }}</h3>
            <p>Precio: ${{ producto.precio }}</p>
        </div>
    {% endfor %}
</div>

Controlador (Controller) - Coordinación

Responsabilidades del Controlador
  • Recibir solicitudes: Capturar peticiones del usuario
  • Coordinar componentes: Llamar métodos apropiados del modelo
  • Procesar datos: Preparar información para la vista
  • Gestionar flujo: Dirigir navegación y respuestas
Ejemplo en Flask:
from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def mostrar_productos():
    # Obtener datos del modelo
    productos = [
        Producto("Laptop", 1200), 
        Producto("Teléfono", 800)
    ]
    # Pasar datos a la vista
    return render_template("productos.html", productos=productos)

if __name__ == '__main__':
    app.run(debug=True)

Comparación entre MVC y Otros Patrones

MVC se diferencia de otros patrones arquitectónicos por su enfoque específico en aplicaciones interactivas.

MVC vs Diseño en Capas

Patrón MVC
  • Estructura: Modelo, Vista, Controlador
  • Flujo: Controlador dirige información
  • Enfoque: Interacción con usuario
  • Uso común: Aplicaciones web y escritorio
Diseño en Capas
  • Estructura: Múltiples capas funcionales
  • Flujo: Comunicación secuencial entre capas
  • Enfoque: Lógica de negocio compleja
  • Uso común: Sistemas empresariales distribuidos
CAPAS TRADICIONALES:
Presentación → Lógica Negocio → Acceso Datos → Base Datos

MVC:
Usuario → Controlador ↔ Modelo
              ↓         ↓
            Vista ←─────┘

Ventajas frente a Arquitecturas Monolíticas

Problemas de Arquitecturas Monolíticas

  • Dificultad mantenimiento: Modificar una parte afecta otras
  • Código desorganizado: Lógica mezclada con interfaz
  • Problemas reutilización: No se pueden reutilizar componentes
  • Acoplamiento alto: Dependencias complejas entre módulos

MVC soluciona estos problemas mediante separación clara de responsabilidades.

Implementación Práctica del MVC en Python

Python ofrece múltiples frameworks para implementar MVC, desde enfoques ligeros hasta soluciones completas.

Frameworks Python para MVC

Python proporciona diversas opciones según complejidad y necesidades del proyecto:

  • Flask: Ligero, flexible, proyectos pequeños
  • FastAPI: Moderno, APIs rápidas, asyncio
  • Django: Completo, robusto, proyectos grandes
  • Pyramid: Intermedio, configuración flexible

Ejemplo MVC Básico en Python Puro

Implementación sin framework:
# MODELO - Gestión de datos
class ProductoModel:
    def __init__(self):
        self.productos = []
    
    def agregar_producto(self, nombre, precio):
        producto = {"nombre": nombre, "precio": precio}
        self.productos.append(producto)
        return producto
    
    def obtener_productos(self):
        return self.productos

# VISTA - Presentación
class ProductoView:
    def mostrar_productos(self, productos):
        print("=== LISTA DE PRODUCTOS ===")
        for i, producto in enumerate(productos, 1):
            print(f"{i}. {producto['nombre']} - ${producto['precio']}")
    
    def mostrar_menu(self):
        print("\n1. Ver productos")
        print("2. Agregar producto")
        print("3. Salir")

# CONTROLADOR - Coordinación
class ProductoController:
    def __init__(self):
        self.model = ProductoModel()
        self.view = ProductoView()
    
    def ejecutar(self):
        while True:
            self.view.mostrar_menu()
            opcion = input("Seleccione opción: ")
            
            if opcion == "1":
                productos = self.model.obtener_productos()
                self.view.mostrar_productos(productos)
            elif opcion == "2":
                nombre = input("Nombre: ")
                precio = float(input("Precio: "))
                self.model.agregar_producto(nombre, precio)
            elif opcion == "3":
                break

# Uso
controller = ProductoController()
controller.ejecutar()

Flask: Enfoque Ligero para Aplicaciones Pequeñas

Flask es un microframework que proporciona flexibilidad máxima para implementar MVC según necesidades específicas.

Características Principales de Flask

Ligero y Minimalista
  • Núcleo pequeño y extensible
  • Solo incluye lo esencial
  • Agregar funcionalidades según necesidad
Flexibilidad Total
  • Control completo sobre arquitectura
  • Elección libre de componentes
  • Configuración personalizada
Estructura MVC en Flask:
# app.py - Aplicación principal
from flask import Flask, render_template, request, redirect, url_for

app = Flask(__name__)

# MODELO - Gestión de datos
class ProductoModel:
    productos = []
    
    @classmethod
    def agregar(cls, nombre, precio):
        producto = {"id": len(cls.productos) + 1, "nombre": nombre, "precio": precio}
        cls.productos.append(producto)
        return producto
    
    @classmethod
    def obtener_todos(cls):
        return cls.productos

# CONTROLADOR - Rutas y lógica de control
@app.route('/')
def index():
    productos = ProductoModel.obtener_todos()
    return render_template('productos.html', productos=productos)

@app.route('/agregar', methods=['GET', 'POST'])
def agregar_producto():
    if request.method == 'POST':
        nombre = request.form['nombre']
        precio = float(request.form['precio'])
        ProductoModel.agregar(nombre, precio)
        return redirect(url_for('index'))
    
    return render_template('agregar.html')

if __name__ == '__main__':
    app.run(debug=True)

Cuándo Usar Flask

Ideal para:
  • Aplicaciones pequeñas y medianas
  • APIs REST simples
  • Prototipos rápidos
  • Microservicios
  • Proyectos con requisitos específicos
  • Aprendizaje de conceptos MVC

Django: Enfoque Estructurado con Herramientas

Django es un framework completo que implementa MVC (llamado MTV: Model-Template-View) con herramientas integradas.

Características Principales de Django

Batteries Included
  • ORM integrado para base de datos
  • Sistema de autenticación
  • Panel de administración automático
  • Sistema de templates robusto
Seguridad y Escalabilidad
  • Protección CSRF automática
  • Validación de datos integrada
  • Migraciones de BD automáticas
  • Escalable para aplicaciones grandes
Estructura MVC en Django:
# models.py - MODELO
from django.db import models

class Producto(models.Model):
    nombre = models.CharField(max_length=100)
    precio = models.DecimalField(max_digits=10, decimal_places=2)
    
    def calcular_descuento(self, porcentaje):
        return float(self.precio) * (1 - porcentaje / 100)

# views.py - CONTROLADOR
from django.shortcuts import render
from .models import Producto

def lista_productos(request):
    productos = Producto.objects.all()  # Consulta al modelo
    return render(request, 'productos.html', {'productos': productos})

# urls.py - Configuración de rutas
from django.urls import path
from . import views

urlpatterns = [
    path('productos/', views.lista_productos, name='productos'),
]

Cuándo Usar Django

Ideal para:
  • Aplicaciones web complejas
  • Sistemas con muchos modelos de datos
  • Proyectos que requieren panel admin
  • E-commerce y CMS
  • Aplicaciones empresariales
  • Proyectos con equipos grandes

Django MTV vs MVC Tradicional

Django usa Model-Template-View (MTV):

  • Model: Equivale al Model de MVC
  • Template: Equivale a la View de MVC (presentación)
  • View: Equivale al Controller de MVC (lógica de control)
