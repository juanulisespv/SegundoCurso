Diseño nativo para plataformas móviles - Parte 1

Nota: Este tema está dividido en 2 partes debido a la extensión del contenido.

1. Introducción

El diseño de interfaces móviles nativas para Android e iOS requiere comprensión profunda de las convenciones, patrones y filosofías únicas de cada plataforma. Cada sistema operativo móvil ha desarrollado su propio lenguaje de diseño, principios de interacción y expectativas de usuario.

Concepto Central: "Diseñar para la plataforma, no contra ella. Los usuarios esperan que las apps se sientan nativas a su dispositivo."
Android Philosophy
Principios Fundamentales
  • Enchant me: Belleza, simplicidad y poder en harmonía
  • Simplify my life: Interfaces que simplifican tareas complejas
  • Make me amazing: Empoderar usuarios para lograr más
Características Distintivas
  • Customización: Usuarios pueden personalizar profundamente
  • Diversidad: Múltiples OEMs, tamaños, especificaciones
  • Openness: Sistema abierto, múltiples entry points
  • Integration: Servicios Google profundamente integrados
iOS Philosophy
Principios Fundamentales
  • Clarity: Texto legible, iconos precisos, funcionalidad evidente
  • Deference: UI fluido que no compite con contenido
  • Depth: Capas visuales que imitan objetos reales
Características Distintivas
  • Consistencia: Experiencia unificada en todos los dispositivos
  • Simplicity: Enfoque en funcionalidad esencial
  • Quality: Estándares altos de fit and finish
  • Innovation: Pionero en gestos y interacciones
Importancia del Diseño Nativo

Las aplicaciones que siguen las convenciones de cada plataforma tienen 67% mayor probabilidad de ser bien valoradas por usuarios y 40% mejor retención que apps que usan patrones cross-platform genéricos.

3. Android Design Fundamentals

El diseño para Android se basa en Material Design, pero también incluye principios únicos de flexibilidad, customización y diversidad de hardware que definen la experiencia Android.

Architecture de App Android
Estructura Típica de Pantalla Android
Status Bar - Notificaciones, hora, batería
App Bar / Toolbar - Título, acciones, navegación
Content Area
• Fragments para diferentes secciones
• RecyclerView para listas
• ViewPager para tabs
• FAB para acción principal
Navigation Bar - Back, Home, Recent Apps
Patrones de Navegación Android
Back Navigation
  • Hardware back button: Universal en Android
  • Up navigation: Dentro de la app hierarchy
  • Predictive back: Preview del destino (Android 13+)
Navigation Drawer
  • Hamburger menu: Acceso a secciones principales
  • Account switching: Múltiples cuentas
  • Settings access: Configuración rápida
Android Considerations
Hardware Variations
  • Screen densities: LDPI a XXXHDPI
  • Aspect ratios: 16:9, 18:9, 19.5:9, 21:9
  • Physical controls: Capacitive vs hardware buttons
  • Notches/cutouts: Variedad de implementaciones
Performance Considerations
  • Memory management: Automatic lifecycle
  • Battery optimization: Doze mode, app standby
  • Network awareness: Metered connections
Accessibility Features
  • TalkBack: Screen reader integrado
  • Switch Access: Para usuarios con discapacidades motoras
  • Magnification: Zoom gestures
  • High contrast: Mejores para low vision

5. Material Design System

Material Design es el lenguaje de diseño de Google que combina principios clásicos de buen diseño con innovación en tecnología y ciencia. Está fundamentado en la metáfora de materiales físicos.

Filosofía Material Design

"Crear un lenguaje visual que sintetiza principios clásicos de buen diseño con innovación y posibilidad de la tecnología y ciencia."

Principios Fundamentales
1. Material as Metaphor

Las superficies digitales imitan las propiedades físicas de papel y tinta, pero son tecnológicamente avanzadas.

  • Surfaces: Tienen thickness (1dp) y cast shadows
  • Elevation: Z-axis para hierarchy y focus
  • Motion: Responde a user input con realistic physics
  • Environment: 3D space con light sources
2. Bold, Graphic, Intentional

Elementos de diseño print - typography, grids, space, scale, color, imagery - guían visual treatments.

  • Color: Deliberate color choices
  • Typography: Roboto family optimization
  • Imagery: Large, heroic images
  • Whitespace: Creates focus y hierarchy
3. Motion Provides Meaning

Motion respeta y refuerza el usuario como primary initiator de acción.

  • Responsive interaction: Acknowledges input
  • Natural easing: Realistic acceleration/deceleration
  • Choreography: Coordinated element transitions
  • Transformation: Seamless object morphing
Material Design 3 (Material You)
Dynamic Color System

Material You introduce personalización automática basada en wallpaper del usuario.

Color Tokens
  • Primary: Accent color principal
  • Secondary: Complementary accents
  • Tertiary: Contrasting accents
  • Neutral: Surfaces y outlines
  • Neutral Variant: Medium emphasis
  • Error: Estados de error
Adaptive: Cada color tiene variants para light/dark themes
Shape System
Shape Categories
  • Small components: 4dp radius (buttons, chips)
  • Medium components: 8dp radius (cards, text fields)
  • Large components: 16dp radius (sheets, dialogs)
  • Extra large: 28dp radius (key surfaces)
Shape Tokens
  • Corner family: Rounded, cut, full-round
  • Corner size: None, extra-small a extra-large
Material Components Ecosystem
Navigation Components
  • App bars: Top y bottom app bars
  • Navigation drawer: Side navigation
  • Navigation rail: Side navigation para large screens
  • Tabs: Fixed y scrollable tabs
  • Bottom navigation: 3-5 top-level destinations
Action Components
  • Buttons: Text, outlined, filled
  • FAB: Floating Action Button
  • Extended FAB: Con text label
  • Icon buttons: Toggle y standard
  • Chips: Compact elements para actions
Input Components
  • Text fields: Filled y outlined variants
  • Checkboxes: Multi-selection
  • Radio buttons: Single selection
  • Switches: Binary states
  • Sliders: Range selection

6. Human Interface Guidelines (Apple)

Las Human Interface Guidelines (HIG) de Apple establecen los principios y mejores prácticas para crear experiencias exceptionales en todos los dispositivos Apple.

iOS Design Principles
1. Clarity (Claridad)

Todo el contenido debe ser legible y comprensible en cualquier tamaño.

  • Text legibility: SF Pro typeface optimizada
  • Icon clarity: Symbols universalmente reconocibles
  • Functionality obviousness: Elementos activos claramente distinguibles
  • Negative space: Whitespace que mejora comprehension
2. Deference (Deferencia)

La UI debe ayudar a los usuarios entender e interactuar con contenido, pero nunca competir con él.

  • Content priority: UI supports, no overshadows content
  • Subtle visual indicators: Gentle cues para functionality
  • Appropriate emphasis: Important elements stand out
  • Minimalism: Solo elementos necesarios
3. Depth (Profundidad)

Layering y motion crean hierarchy, vitality y understanding.

  • Z-axis layering: Establece information hierarchy
  • Parallax effects: Depth perception durante scroll
  • Blur effects: Background context preservation
  • Shadows y highlights: Physical material simulation
