7. Gestos e Interacciones

Los gestos táctiles son fundamentales en interfaces móviles. Cada plataforma ha desarrollado su propio vocabulario gestual que los usuarios aprenden y esperan encontrar consistentemente.

Gestos Android
Tap & Long Press

Tap: Activar, seleccionar

Long Press: Context menu, drag mode

Swipe Gestures

Horizontal Swipe: Navegación entre tabs, dismiss notifications

Vertical Swipe: Scroll, pull-to-refresh, quick settings

System Gestures (Android 10+)
  • Swipe up from bottom: Home
  • Swipe up and hold: Recent apps
  • Swipe from edges: Back navigation
  • Diagonal swipe: Google Assistant
Gestos iOS
Basic Touch Interactions

Tap: Select, activate

Force Touch: Peek & Pop (legacy), contextual actions

Navigation Gestures

Edge Swipe: Back navigation (desde iOS 7)

Pull Down: Control Center, Notification Center

System Gestures (iPhone X+)
  • Swipe up from bottom: Home
  • Swipe up and pause: App switcher
  • Swipe down from top-right: Control Center
  • Swipe down from top-left: Notifications
  • Swipe bottom edge: Quick app switching
Gesture Accessibility

Siempre proporciona alternativas para gestos complejos. Algunos usuarios no pueden realizar multi-touch gestures o swipes precisos. Include botones alternativos y voice commands cuando sea posible.

9. Componentes Nativos

Cada plataforma proporciona un conjunto de componentes nativos optimizados para performance y consistencia con el sistema operativo.

Función Android Component iOS Component Diferencias Clave

Lists RecyclerView UITableView Android: flexible layouts; iOS: structured sections
Grids RecyclerView + GridLayoutManager UICollectionView Android: adaptive span count; iOS: flow layouts
Alerts AlertDialog UIAlertController Android: más customizable; iOS: system consistency
Action Sheets BottomSheetDialog UIActionSheet/UIAlertController Android: modal bottom sheet; iOS: platform-specific presentation
Switches Switch/ToggleButton UISwitch Android: multiple styles; iOS: single consistent design

Android Specific Components
  • Floating Action Button (FAB): Primary action prominente
  • Snackbar: Lightweight feedback y undo actions
  • Navigation View: Drawer menu structure
  • CoordinatorLayout: Complex scrolling behaviors
  • BottomSheet: Modal content desde bottom
  • Chips: Compact elements para filters/actions
  • App Shortcuts: Long-press home screen quick actions
iOS Specific Components
  • Navigation Controller: Hierarchical navigation stack
  • Segmented Control: Mutually exclusive choices
  • Action Sheet: Contextual action menu
  • Popover: iPad contextual content
  • Activity Indicator: Loading states
  • Stepper: Increment/decrement values
  • 3D Touch Actions: Force-sensitive quick actions

10. Adaptabilidad y Responsive Design

Las interfaces móviles deben adaptarse dinámicamente a diferentes tamaños de pantalla, orientaciones y contextos de uso. Cada plataforma maneja esto de manera diferente.

Android Adaptive Design
Configuration Qualifiers

Android usa resource qualifiers para proporcionar layouts alternativos.

Screen Size Qualifiers
  • small: ≥426dp x 320dp
  • normal: ≥470dp x 320dp
  • large: ≥640dp x 480dp
  • xlarge: ≥960dp x 720dp
Density Qualifiers
  • ldpi: ~120dpi (0.75x)
  • mdpi: ~160dpi (1x baseline)
  • hdpi: ~240dpi (1.5x)
  • xhdpi: ~320dpi (2x)
  • xxhdpi: ~480dpi (3x)
  • xxxhdpi: ~640dpi (4x)
Fragment-Based Architecture

Fragments permiten interfaces modulares que se recombinan según screen size.

Phone Layout
📱 Single pane: List Fragment → Detail Fragment (navigation)
Tablet Layout
📱 Dual pane: List Fragment | Detail Fragment (side-by-side)
iOS Adaptive Design
Size Classes

iOS usa size classes abstractas que combinan width y height.

Horizontal Size Classes
  • Compact: iPhone portrait, iPhone landscape (smaller models)
  • Regular: iPad any orientation, iPhone Plus/Max landscape
Vertical Size Classes
  • Compact: iPhone landscape
  • Regular: iPhone portrait, iPad any orientation

11. Casos de Estudio

Éxito: WhatsApp Cross-Platform Consistency
Desafío

Mantener experiencia familiar entre plataformas mientras respeta convenciones nativas.

Estrategia de Diseño
  • Core experience: Chat interface consistent
  • Platform adaptations: Navigation patterns nativos
  • Visual consistency: Green color scheme universal
  • Feature parity: Misma funcionalidad en ambas platforms
Platform-Specific Adaptations
Android
  • FAB para nueva conversación
  • Tabs en top para Chats/Status/Calls
  • Material Design cards
  • System back button support
iOS
  • Tab bar en bottom
  • Navigation controller stack
  • iOS-style list cells
  • Swipe gestures para actions
Resultado: 95% user satisfaction en ambas plataformas, +2B usuarios activos
Caso: Instagram Stories Innovation
Innovation Challenge

Crear nuevo pattern de content consumption que se sienta nativo en ambas plataformas.

Design Solutions
  • Gesture-first: Tap para next, swipe para dismiss
  • Full-screen immersion: Content-focused experience
  • Temporal UI: Progress indicators y auto-advance
  • Quick interactions: React, reply, share sin interrumpir flow
Platform Optimizations
Android
  • Hardware back button exits stories
  • Volume keys para navigation
  • Share sheet integration
iOS
  • 3D Touch para preview
  • iOS share sheet
  • Haptic feedback
Impacto: Pattern adoptado por múltiples platforms (Snapchat, Facebook, YouTube)

12. Resumen y Best Practices

Principios Universales para Interfaces Móviles
Diseño
  • Respeta las convenciones de cada plataforma
  • Prioriza contenido sobre decoración UI
  • Diseña para thumb-friendly interactions
  • Considera contextos de uso real (luz, movimiento, distracciones)
Implementación
  • Usa componentes nativos cuando sea posible
  • Optimiza para performance en hardware limitado
  • Implementa accessibility desde el inicio
  • Testa en dispositivos reales, no solo simuladores
Best Practices
  • Touch targets: Mínimo 44x44pt (iOS) / 48x48dp (Android)
  • Text legibility: Mínimo 16px en mobile, high contrast
  • Loading states: Skeleton screens, progressive loading
  • Error handling: Graceful degradation, clear recovery paths
  • Offline support: Cache content crítico, estados offline claros
  • Battery awareness: Minimize background processing
Anti-Patterns
  • Tiny touch targets: Botones <44pt causan frustración
  • Platform confusion: Android patterns en iOS y viceversa
  • Over-customization: Reinventar componentes estándar
  • Ignoring accessibility: No support para screen readers
  • Poor performance: Animations janky, loading lento
  • Inconsistent navigation: Back button behavior impredecible
Flujo de Desarrollo Mobile-First
Research
Platform guidelines, user behavior patterns
Mobile Design
Start with smallest screen, core functionality
Scale Up
Enhance for larger screens, add features
Test & Iterate
Real devices, real users, real contexts
